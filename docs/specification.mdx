## ディレクトリ構成

スキルは、最低限 `SKILL.md` ファイルを含むディレクトリです。

```
skill-name/
├── SKILL.md          # 必須: メタデータ + 手順
├── scripts/          # 任意: 実行可能コード
├── references/       # 任意: ドキュメント
├── assets/           # 任意: テンプレート、リソース
└── ...               # 追加のファイル/ディレクトリ
```

## `SKILL.md` フォーマット

`SKILL.md` ファイルには、YAML frontmatter とそれに続く Markdown 本文が必要です。

### Frontmatter

| フィールド      | 必須   | 制約                                                                                  |
| --------------- | ------ | ------------------------------------------------------------------------------------- |
| `name`          | はい   | 最大64文字。小文字、数字、ハイフンのみ。先頭/末尾にハイフン不可。                     |
| `description`   | はい   | 最大1024文字。空文字不可。スキルの内容と使いどころを記述。                            |
| `license`       | いいえ | ライセンス名、または同梱ライセンスファイルへの参照。                                  |
| `compatibility` | いいえ | 最大500文字。環境要件（対象製品、システムパッケージ、ネットワークアクセス等）を示す。 |
| `metadata`      | いいえ | 追加メタデータ用の任意キー/値マッピング。                                             |
| `allowed-tools` | いいえ | 事前承認済みツールのスペース区切り文字列。（実験的）                                  |

<Card>
**最小例:**

```markdown SKILL.md
---
name: skill-name
description: A description of what this skill does and when to use it.
---
```

**任意フィールドを含む例:**

```markdown SKILL.md
---
name: pdf-processing
description: Extract PDF text, fill forms, merge files. Use when handling PDFs.
license: Apache-2.0
metadata:
  author: example-org
  version: "1.0"
---
```

</Card>

#### `name` フィールド

必須の `name` フィールド:

- 1〜64文字であること
- Unicode の小文字英数字（`a-z`, `0-9`）とハイフン（`-`）のみ利用可能
- 先頭/末尾をハイフン（`-`）にできない
- 連続ハイフン（`--`）を含めない
- 親ディレクトリ名と一致している

<Card>
**有効な例:**
```yaml
name: pdf-processing
```
```yaml
name: data-analysis
```
```yaml
name: code-review
```

**無効な例:**

```yaml
name: PDF-Processing # 大文字は不可
```

```yaml
name: -pdf # 先頭ハイフン不可
```

```yaml
name: pdf--processing # 連続ハイフン不可
```

</Card>

#### `description` フィールド

必須の `description` フィールド:

- 1〜1024文字であること
- スキルが何をするかと、いつ使うかの両方を記述する
- 関連タスクをエージェントが判定しやすい具体的キーワードを含める

<Card>
**良い例:**
```yaml
description: Extracts text and tables from PDF files, fills PDF forms, and merges multiple PDFs. Use when working with PDF documents or when the user mentions PDFs, forms, or document extraction.
```

**悪い例:**

```yaml
description: Helps with PDFs.
```

</Card>

#### `license` フィールド

任意の `license` フィールド:

- スキルに適用されるライセンスを指定する
- 短く記述することを推奨（ライセンス名、または同梱ライセンスファイル名）

<Card>
  **例:** ```yaml license: Proprietary. LICENSE.txt has complete terms ```
</Card>

#### `compatibility` フィールド

任意の `compatibility` フィールド:

- 指定する場合は1〜500文字
- スキルに特定の環境要件がある場合のみ記述を推奨
- 対象製品、必要なシステムパッケージ、ネットワークアクセス要件などを示せる

<Card>
  **例:** ```yaml compatibility: Designed for Claude Code (or similar products)
  ``` ```yaml compatibility: Requires git, docker, jq, and access to the
  internet ``` ```yaml compatibility: Requires Python 3.14+ and uv ```
</Card>

<Note>多くのスキルでは `compatibility` フィールドは不要です。</Note>

#### `metadata` フィールド

任意の `metadata` フィールド:

- 文字列キーから文字列値へのマップ
- Agent Skills 仕様で未定義の追加プロパティを保存するためにクライアントが利用可能
- 偶発的な衝突を避けるため、キー名はある程度ユニークにすることを推奨

<Card>**例:** ```yaml metadata: author: example-org version: "1.0" ```</Card>

#### `allowed-tools` フィールド

任意の `allowed-tools` フィールド:

- 実行を事前承認するツールをスペース区切りで列挙
- 実験的機能のため、エージェント実装ごとにサポート状況は異なる

<Card>**例:** ```yaml allowed-tools: Bash(git:*) Bash(jq:*) Read ```</Card>

### 本文

frontmatter の後に続く Markdown 本文にスキル手順を記述します。本文の形式制約はありません。エージェントがタスクを効果的に実行できる内容を書いてください。

推奨セクション:

- ステップごとの手順
- 入力と出力の例
- よくあるエッジケース

エージェントはスキル有効化を決めると、このファイル全体を読み込みます。`SKILL.md` が長い場合は、参照ファイルへ分割することを検討してください。

## 任意ディレクトリ

### `scripts/`

エージェントが実行可能なコードを格納します。スクリプトは次を満たすべきです。

- 単体で動く、または依存関係を明確に記述する
- 役立つエラーメッセージを含む
- エッジケースを適切に処理する

対応言語はエージェント実装に依存します。一般的には Python、Bash、JavaScript が使われます。

### `references/`

必要時にエージェントが読める補助ドキュメントを格納します。

- `REFERENCE.md` - 詳細な技術リファレンス
- `FORMS.md` - フォームテンプレートや構造化データ形式
- ドメイン固有ファイル（`finance.md`, `legal.md` など）

個々の [参照ファイル](#file-references) は焦点を絞ってください。エージェントは必要時に読み込むため、ファイルが小さいほどコンテキスト消費を抑えられます。

### `assets/`

静的リソースを格納します。

- テンプレート（文書テンプレート、設定テンプレート）
- 画像（図、例）
- データファイル（ルックアップテーブル、スキーマ）

## 段階的開示

エージェントはスキルを _段階的_ に読み込み、タスクに応じて必要な詳細だけ取り込みます。スキルはこの性質を活かして構成してください。

1. **Metadata**（約100トークン）: 起動時に全スキルの `name` と `description` を読み込む
2. **Instructions**（推奨 5000 トークン未満）: スキル有効化時に `SKILL.md` 本文全体を読み込む
3. **Resources**（必要時）: `scripts/`、`references/`、`assets/` などのファイルを必要時のみ読み込む

メインの `SKILL.md` は500行未満を目安にし、詳細な資料は別ファイルへ分離してください。

## ファイル参照

スキル内で他ファイルを参照する場合は、スキルルートからの相対パスを使います。

```markdown SKILL.md
See [the reference guide](references/REFERENCE.md) for details.

Run the extraction script:
scripts/extract.py
```

`SKILL.md` からの参照は1段階程度に保ち、深い入れ子の参照チェーンは避けてください。

## 検証

[skills-ref](https://github.com/agentskills/agentskills/tree/main/skills-ref) 参照ライブラリを使ってスキルを検証できます。

```bash
skills-ref validate ./my-skill
```

これにより、`SKILL.md` の frontmatter が有効で、命名規約に従っていることを確認できます。
