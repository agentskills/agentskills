# Skill Crystallisation

**Crystallisation** is the process of transforming natural language skills into coded implementations as their scope becomes clear and edge cases are identified.

<Info>
Skills don't have to stay as prompts forever. As you learn what works, crystallise them into code for determinism, testability, and performance.
</Info>

## The Crystallisation Spectrum

Skills exist on a spectrum from fully natural language to fully coded:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     THE CRYSTALLISATION SPECTRUM                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  FLUID                                                      CRYSTALLISED │
│  (Natural Language)                                              (Code)  │
│                                                                          │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │  Pure    │───▶│  Hybrid  │───▶│  Guided  │───▶│  Pure    │          │
│  │  Prompt  │    │  Prompt  │    │   Code   │    │   Code   │          │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘          │
│                                                                          │
│  • Exploratory   • Structured     • Code with    • Deterministic        │
│  • Flexible      • Validated      • LLM fallback • Testable             │
│  • Adaptable     • Constrained    • Best of both • Performant           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Stage 3: Guided Code (Hybrid)

```python
# Code handles known patterns, LLM handles edge cases
# skill: extract-invoice-data
# classification: READ

import re
from datetime import datetime
from skills import skill, llm_fallback

@skill(classification="READ")
def extract_invoice_data(document_text: str, document_type: str) -> dict:
    """Extract invoice data - crystallised from 6 months of production use."""

    # Known vendor patterns (crystallised knowledge)
    vendor_patterns = {
        r"Amazon Web Services": "AWS",
        r"Google Cloud Platform": "GCP",
        r"Microsoft Azure": "Azure",
        # ... 50+ patterns learned from production
    }

    # Try deterministic extraction first
    result = {
        "vendor": extract_vendor(document_text, vendor_patterns),
        "amount": extract_amount(document_text),
        "currency": extract_currency(document_text),
        "date": extract_date(document_text),
        "line_items": extract_line_items(document_text),
    }

    # Validate completeness
    missing = [k for k, v in result.items() if v is None]

    if missing:
        # Fall back to LLM for edge cases
        return llm_fallback(
            document_text=document_text,
            partial_result=result,
            missing_fields=missing,
            prompt="Complete the missing invoice fields: {missing_fields}"
        )

    return result

def extract_amount(text: str) -> float | None:
    """Crystallised amount extraction - handles 95% of cases."""
    patterns = [
        r"Total:?\s*[\$£€]?([\d,]+\.?\d*)",
        r"Amount Due:?\s*[\$£€]?([\d,]+\.?\d*)",
        r"Grand Total:?\s*[\$£€]?([\d,]+\.?\d*)",
    ]
    for pattern in patterns:
        if match := re.search(pattern, text, re.IGNORECASE):
            return float(match.group(1).replace(",", ""))
    return None
```

### Stage 4: Pure Code (Crystallised)

```python
# Fully deterministic - no LLM required
# skill: extract-aws-invoice
# classification: READ
# crystallised_from: extract-invoice-data
# crystallisation_date: 2025-03-15

from dataclasses import dataclass
from decimal import Decimal
import re

@dataclass
class AWSInvoice:
    account_id: str
    billing_period: str
    total_amount: Decimal
    currency: str
    services: list[dict]

@skill(classification="READ", deterministic=True)
def extract_aws_invoice(document_text: str) -> AWSInvoice:
    """
    100% deterministic AWS invoice extraction.

    Crystallised after processing 10,000+ AWS invoices with:
    - 99.7% accuracy on known formats
    - 0 LLM calls required
    - 50ms average execution (vs 2s with LLM)
    """

    account_id = re.search(r"Account:\s*(\d{12})", document_text).group(1)
    billing_period = re.search(r"Billing Period:\s*(.+)", document_text).group(1)

    # Parse known AWS format
    total_match = re.search(r"Total:\s*\$?([\d,]+\.\d{2})", document_text)
    total_amount = Decimal(total_match.group(1).replace(",", ""))

    # Extract service breakdown
    services = parse_aws_service_lines(document_text)

    return AWSInvoice(
        account_id=account_id,
        billing_period=billing_period,
        total_amount=total_amount,
        currency="USD",
        services=services,
    )
```

## When to Crystallise

Crystallisation is appropriate when:

| Criterion | Threshold | Why It Matters |
|-----------|-----------|----------------|
| **Volume** | 100+ executions | Enough data to identify patterns |
| **Stability** | <5% prompt changes/month | Requirements have settled |
| **Edge cases** | 95%+ documented | Know what to handle |
| **Error rate** | <2% with current prompt | Prompt is already reliable |
| **Performance need** | Latency matters | Code is 10-100x faster |

### Crystallisation Decision Matrix

## The Crystallisation Process

### Step 1: Gather Evidence

```python
# Track skill execution patterns
class CrystallisationMetrics:
    def __init__(self, skill_name: str):
        self.skill_name = skill_name
        self.executions = []
        self.input_patterns = Counter()
        self.output_patterns = Counter()
        self.edge_cases = []
        self.failures = []

    def record_execution(self, input_data: dict, output_data: dict, success: bool):
        self.executions.append({
            "timestamp": datetime.now(),
            "input_hash": self._pattern_hash(input_data),
            "output_hash": self._pattern_hash(output_data),
            "success": success,
        })

        # Track patterns
        self.input_patterns[self._pattern_hash(input_data)] += 1
        self.output_patterns[self._pattern_hash(output_data)] += 1

    def crystallisation_readiness(self) -> dict:
        """Assess if skill is ready for crystallisation."""
        total = len(self.executions)
        if total < 100:
            return {"ready": False, "reason": f"Only {total} executions, need 100+"}

        # Check pattern concentration
        top_5_patterns = sum(c for _, c in self.input_patterns.most_common(5))
        pattern_coverage = top_5_patterns / total

        # Check failure rate
        failures = sum(1 for e in self.executions if not e["success"])
        failure_rate = failures / total

        return {
            "ready": pattern_coverage > 0.8 and failure_rate < 0.02,
            "executions": total,
            "pattern_coverage": pattern_coverage,
            "failure_rate": failure_rate,
            "top_patterns": self.input_patterns.most_common(10),
            "edge_cases": len(self.edge_cases),
        }
```

### Step 2: Identify Core Logic

Extract the deterministic parts from successful executions:

```python
# Analyze what the LLM consistently does
def analyze_skill_outputs(skill_name: str, executions: list[dict]) -> dict:
    """Find patterns in successful skill outputs."""

    patterns = {
        "transformations": [],  # Input→Output mappings
        "validations": [],      # Rules that are always applied
        "edge_cases": [],       # Exceptions that needed special handling
    }

    for execution in executions:
        if execution["success"]:
            # What transformation was applied?
            patterns["transformations"].append(
                diff_transform(execution["input"], execution["output"])
            )

    # Find common transformations
    common = find_common_patterns(patterns["transformations"])

    return {
        "crystallisable": common,
        "requires_llm": patterns["edge_cases"],
        "coverage": len(common) / len(executions),
    }
```

    - v5 (2025-12): 95% crystallised, LLM for edge cases only
    """

    # Try crystallised extraction
    try:

## Maintaining Hybrid Systems

Most production systems will have a mix of fluid and crystallised skills:

```python
# skills/router.py
class SkillRouter:
    """Routes execution between crystallised and prompt versions."""

    def __init__(self):
        self.crystallised_skills = load_crystallised_skills()
        self.prompt_skills = load_prompt_skills()
        self.metrics = CrystallisationMetrics()

    def execute(self, skill_name: str, input_data: dict) -> dict:
        # Check if crystallised version exists and applies
        if skill_name in self.crystallised_skills:
            crystallised = self.crystallised_skills[skill_name]

            if crystallised.can_handle(input_data):
                result = crystallised.execute(input_data)
                self.metrics.record_crystallised_hit(skill_name)
                return result

        # Fall back to prompt version
        prompt_skill = self.prompt_skills[skill_name]
        result = prompt_skill.execute(input_data)

        # Record for future crystallisation
        self.metrics.record_prompt_execution(skill_name, input_data, result)

        return result

    def get_crystallisation_candidates(self) -> list[dict]:
        """Identify skills ready for crystallisation."""
        candidates = []

        for skill_name, metrics in self.metrics.skill_metrics.items():
            readiness = metrics.crystallisation_readiness()
            if readiness["ready"]:
                candidates.append({
                    "skill": skill_name,
                    "executions": readiness["executions"],
                    "pattern_coverage": readiness["pattern_coverage"],
                    "estimated_savings": self._estimate_savings(skill_name),
                })

        return sorted(candidates, key=lambda x: x["estimated_savings"], reverse=True)
```

## Crystallisation Anti-Patterns

<Warning>
Avoid these common crystallisation mistakes.
</Warning>

### ❌ Premature Crystallisation

```python
# BAD: Crystallising after 50 executions
# Not enough data to identify edge cases

@skill(deterministic=True)  # Too early!
def process_invoice(text: str) -> dict:
    # Only handles the 3 invoice formats seen so far
    # Will fail on the 4th format
    ...
```

### ❌ All-or-Nothing Crystallisation

```python
# BAD: Trying to crystallise everything at once
def fully_crystallised_workflow(data):
    # 500 lines of brittle code
    # No fallback for edge cases
    # Fails catastrophically on unexpected input
    ...
```

### ❌ Ignoring the Tail

```python
# BAD: Only crystallising the common cases
@skill(deterministic=True)
def extract_amount(text: str) -> float:
    # Handles 80% of cases perfectly
    # Silently returns wrong values for 20%
    return regex_extract(text)  # No fallback!
```

### ✅ Correct Approach

```python
# GOOD: Gradual crystallisation with fallback
@skill(classification="READ")
def extract_amount(text: str) -> float:
    """Crystallised with LLM fallback for edge cases."""

    # Try crystallised patterns (handles 95% of cases)
    for pattern in CRYSTALLISED_PATTERNS:
        if match := pattern.search(text):
            return parse_amount(match)

    # Log miss for future crystallisation
    log_crystallisation_miss("extract_amount", text)

    # Fall back to LLM for remaining 5%
    return llm_extract_amount(text)
```

## Measuring Crystallisation Success

Track these metrics to evaluate crystallisation effectiveness:

```python
@dataclass
class CrystallisationReport:
    skill_name: str
    period: str  # e.g., "2025-Q1"

    # Coverage
    total_executions: int
    crystallised_hits: int  # Handled by code
    llm_fallbacks: int      # Needed LLM

    # Quality
    accuracy_crystallised: float  # When code handled it
    accuracy_llm: float           # When LLM handled it

    # Performance
    avg_latency_crystallised_ms: float
    avg_latency_llm_ms: float

    # Cost
    llm_cost_before: Decimal
    llm_cost_after: Decimal

    @property
    def crystallisation_rate(self) -> float:
        return self.crystallised_hits / self.total_executions

    @property
    def latency_improvement(self) -> float:
        return self.avg_latency_llm_ms / self.avg_latency_crystallised_ms

    @property
    def cost_savings(self) -> Decimal:
        return self.llm_cost_before - self.llm_cost_after

    def summary(self) -> str:
        return f"""
Crystallisation Report: {self.skill_name} ({self.period})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Coverage:    {self.crystallisation_rate:.1%} handled by code
Latency:     {self.latency_improvement:.0f}x faster
Cost:        ${self.cost_savings:,.2f} saved
Accuracy:    {self.accuracy_crystallised:.1%} (code) vs {self.accuracy_llm:.1%} (LLM)
"""
```

## When NOT to Crystallise

Some skills should remain as prompts:

| Skill Type | Why Keep as Prompt |
|------------|-------------------|
| **Creative tasks** | Novelty is the point |
| **Evolving domains** | Rules change frequently |
| **Low volume** | Not worth the engineering effort |
| **High variance** | No consistent patterns to extract |
| **Judgment calls** | Requires contextual reasoning |

```yaml
# This skill should stay as a prompt
name: write-marketing-copy
type: atomic
classification: WRITE

crystallisation:
  status: not_recommended
  reason: "Creative task with high variance - crystallisation would reduce quality"

  evidence:
    - "Output diversity is a feature, not a bug"
    - "Brand voice evolves quarterly"
    - "A/B tests show LLM copy outperforms templates"
```

## Summary

Skill crystallisation is the natural evolution of mature skills:

1. **Start fluid** - Explore with natural language prompts
2. **Gather evidence** - Track patterns, edge cases, failures
3. **Crystallise gradually** - Code the deterministic parts first
4. **Keep fallbacks** - LLM handles what code cannot
5. **Measure success** - Track coverage, latency, cost savings

The goal is not to eliminate LLMs, but to use them where they add value while crystallising the predictable parts into reliable, fast, testable code.
