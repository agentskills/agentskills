このチュートリアルでは、乱数生成を使ってサイコロを振る能力をエージェントに与えるスキルを作成します。

## 前提条件

- [VS Code](https://code.visualstudio.com/) with [GitHub Copilot](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot)

<Note>
  このチュートリアルは VS Code を使いますが、Agent Skills
  はオープンフォーマットです。同じスキルは Claude Code や OpenAI Codex
  を含む互換エージェントでも動作します。
</Note>

## スキルを作成する

スキルは `SKILL.md` ファイルを含むフォルダです。VS Code はデフォルトで `.agents/skills/` を探索します。プロジェクト内に `.agents/skills/roll-dice/SKILL.md` を作成してください。

````markdown .agents/skills/roll-dice/SKILL.md
---
name: roll-dice
description: Roll dice using a random number generator. Use when asked to roll a die (d6, d20, etc.), roll dice, or generate a random dice roll.
---

To roll a die, use the following command that generates a random number from 1
to the given number of sides:

```bash
echo $((RANDOM % <sides> + 1))
```

```powershell
Get-Random -Minimum 1 -Maximum (<sides> + 1)
```

Replace `<sides>` with the number of sides on the die (e.g., 6 for a standard
die, 20 for a d20).
````

以上で完了です。1ファイル、20行未満で作れます。各要素の意味は次のとおりです。

- **`name`** — スキルの短い識別子。フォルダ名と一致させる必要があります。
- **`description`** — このスキルをいつ使うかをエージェントに伝えます。エージェントはこれを基に有効化するか判断します。
- **本文** — スキル有効化時にエージェントが従う手順です。ここではユーザー要求の面数に置き換えて、ターミナルコマンドで乱数を生成するよう指示しています。

## 試してみる

1. VS Code でプロジェクトを開きます。
2. Copilot Chat パネルを開きます。
3. チャットパネル下部のモード選択から **Agent** モードを選びます。
4. `/skills` を入力し、`roll-dice` が一覧に表示されることを確認します。表示されない場合は、プロジェクトルートから見て `.agents/skills/roll-dice/SKILL.md` にあるか確認してください。
5. **"Roll a d20"** と入力します。

エージェントが `roll-dice` スキルを有効化するはずです。ターミナルコマンド実行の許可を求められたら許可してください。コマンドが実行され、1〜20のランダムな数値が返ります。

<Note>
  ツール実行の安定性はモデルによって異なります。スキル手順に従って一貫してコマンド実行するモデルもあれば、独自回答を試みるモデルもあります。ターミナルコマンドを実行せずに応答する場合は、モデル選択を変更して再試行してください。
</Note>

## 仕組み

内部では次の流れで動作しています。

1. **Discovery** — チャットセッション開始時に、エージェントが既定のスキルディレクトリを走査し、あなたのスキルを発見します。まずは `name` と `description` だけを読み、関連性の判断に必要な最小情報だけ取得します。

2. **Activation** — サイコロを振る依頼をすると、質問がスキル説明に一致し、`SKILL.md` 本文全体がコンテキストに読み込まれます。

3. **Execution** — エージェントが本文の手順に従い、依頼された面数に合わせてターミナルコマンドを調整して実行します。

この仕組みは **段階的開示（progressive disclosure）** により、すべての手順を最初に読み込まずに多数のスキルを扱えるようにしています。
